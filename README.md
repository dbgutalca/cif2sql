<h1>CIF2SQL</h1>
CIF2SQL is a Java-based tool that reads PDB files in CIF format using the CIF-Tools library and generates SQL files containing the instructions required to insert the data into a relational database.
The tool extracts information from categories defined in the PDB Exchange Dictionary (PDBx/mmCIF) Version 5.0:
https://mmcif.wwpdb.org/dictionaries/mmcif_pdbx_v50.dic/Index/index.html
Additionally, the application includes scripts to:

- Download mmCIF files from the PDB repository to a local machine
- Create a PostgreSQL database and its tables
- Insert the generated SQL data into the database

<h1>Requirements</h1>
To use this tool, you need:


•	Java 8
  https://www.java.com/en/download/ie_manual.jsp
  
•	A local copy of mmCIF files.
  A download script is included in the `download-mmcif.sh` file.
  
•	PostgreSQL (recommended).
  CIF2SQL was primarily tested using PostgreSQL as the database system.

<h1>Application download</h1>
The application and all required files are included in the CIF2SQL ZIP package. You can download it using the following link:
  cif2sql.zip

<h1>Directory Structure</h1>
Once downloaded and extracted, the package will contain the following files and directories:

- `db_logs/`
  Contains error logs generated during PostgreSQL data insertion.
- `logs/`
  Contains logs of converted, omitted, and verified files to prevent reprocessing files that are already stored in the database.
- `sql-files/`
  Stores all SQL files generated by CIF2SQL, organized into folders named after each execution.
- `path.txt`
  Contains the absolute path to the directory where the mmCIF files are located.
- `mmcif-downloader.sh`
  Script used to download the complete mmCIF dataset from the official PDB repository.
- `sql-loader.sh`
  Script used to load the generated SQL files into a PostgreSQL database.
- `mmcif2sql.jar`
  Main application file. Run it from a terminal using:
  `java -jar mmcif2sql.jar`
- `dbschema.sql`
  Contains the SQL `CREATE TABLE` instructions required to create the database schema.
  
<h1>Database creation</h1>

To prepare the database, execute the `dbschema.sql` file in your local PostgreSQL database. This script contains the SQL `CREATE TABLE` instructions required to generate the database schema used by CIF2SQL.
Once the schema has been created, the database will be ready to store the SQL data generated from mmCIF protein structure files.

<h1>Downloading mmCIF Files</h1>
There are two ways to obtain mmCIF files:

- Individual download  
  Open an RCSB PDB structure page, for example:
  https://www.rcsb.org/structure/9ZHX
  Then download the structure in `PDBx/mmCIF` format.
- Using the download script  
  The `mmcif-download.sh` script executes an `rsync` command to download all mmCIF files available in the official PDB repository:
  http://rsync.ebi.ac.uk/pub/databases/pdb/data/structures/divided/mmCIF

  The downloaded files are organized into folders and stored in the `cif-files/` directory.
  
<h1>CIF2SQL Transformation</h1>

To use CIF2SQL, first open the `cif-files-path.txt` file and specify the directory where the mmCIF files are stored. Then, open a terminal in the directory where `cif2sql.jar` is located and run:
```bash
java -jar cif2sql.jar
```

After execution, a new folder will be created inside the sql-files/ directory using the execution date as its name. For example: 08-05-2026
During subsequent executions, CIF2SQL checks the processed-files log and compares it against newly downloaded mmCIF files to avoid processing the same files multiple times.
The generated SQL files can then be loaded into the previously created PostgreSQL database.
